# Лабораторная работа 3 — Билатеральная фильтрация на GPU (CUDA)

## Описание

Реализация 9-точечного билатерального фильтра с использованием CUDA и текстурной памяти GPU. Программа обрабатывает полутоновое изображение в формате BMP и сравнивает время выполнения на CPU и GPU.

Билатеральный фильтр — расширение гауссового размытия, которое сглаживает изображение, сохраняя границы объектов. Новое значение интенсивности пикселя вычисляется по формуле:

```
h(a0) = k⁻¹ · Σ f(aᵢ) · g(aᵢ) · r(aᵢ)
```

где:

- `g(x, y) = exp(-(x²+y²) / (2·σd²))` — пространственный вес (зависит от расстояния),
- `r(aᵢ) = exp(-(f(aᵢ)-f(a0))² / (2·σr²))` — диапазонный вес (зависит от разницы яркостей),
- `k` — нормирующая константа.

## Структура проекта

```
.
├── bilateral.cu       # Исходный код (CPU + GPU реализации)
├── stb_image.h        # Библиотека загрузки изображений
├── stb_image_write.h  # Библиотека сохранения изображений
├── image.bmp          # Входное изображение
└── output.bmp         # Результат после фильтрации
```

## Сборка

```bash
nvcc bilateral.cu -o bilateral -O2
```

## Запуск

```bash
./bilateral <input.bmp> <output.bmp> <sigmaD> <sigmaR>
```

**Параметры:**
| Параметр | Описание |
|----------|----------|
| `input.bmp` | Входное полутоновое BMP-изображение |
| `output.bmp` | Путь для сохранения результата |
| `sigmaD` | Параметр пространственного гауссиана (σd) |
| `sigmaR` | Параметр диапазонного фильтра (σr) |

**Пример:**

```bash
./bilateral image.bmp output.bmp 3.0 50.0
```

## Реализация

### CPU-версия

Последовательный перебор всех пикселей изображения. Для граничных пикселей применяется clamp — берётся значение ближайшего крайнего пикселя.

### GPU-версия (CUDA)

- Входное изображение копируется в `cudaArray` и привязывается к объекту текстуры (`cudaTextureObject_t`).
- Каждый пиксель обрабатывается отдельным CUDA-потоком.
- Обращение к соседним пикселям производится через текстурную память с режимом адресации `cudaAddressModeClamp`.
- Блоки потоков: `16×16`, сетка покрывает всё изображение.
- Время выполнения GPU замеряется через `cudaEvent`.

## Пример вывода

```
Image size: 614 x 441

GPU time: 1.83 ms
Output saved to: output.bmp
```

## Результат

Входное изображение после применения билатерального фильтра сохраняет чёткие границы между объектами, при этом шум на однородных поверхностях сглаживается. GPU-ускорение по сравнению с CPU составляет более 100x.
