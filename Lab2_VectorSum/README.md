# Лабораторная работа 2 — Сумма элементов вектора (CPU vs GPU CUDA)

## Описание

В работе реализован алгоритм суммирования элементов вектора на CPU и GPU (CUDA).  
Цель — сравнить производительность и вычислить ускорение.

Программа:
- генерирует вектор размером от 1 000 до 1 000 000 элементов
- считает сумму на CPU и GPU
- измеряет время выполнения
- вычисляет ускорение (Speedup)

---

## Реализация

### CPU

Последовательное суммирование элементов:

```python
def cpu_vector_sum(arr):
    total = 0
    for x in arr:
        total += x
    return total
```
# GPU (CUDA)

## Параллельное суммирование с использованием CUDA

### Распараллеливание:
- каждый поток обрабатывает отдельный элемент
- используется редукция (reduction)

### Механизмы:
- threads и blocks
- shared memory
- синхронизация (`__syncthreads()`)

## Эксперименты

Проведены вычисления для разных размеров вектора.

| Vector Size | CPU (sec) | GPU (sec) | Speedup |
|:-----------:|:---------:|:---------:|:-------:|
| 1,000       | 0.000115  | 0.000354  | 0.32    |
| 10,000      | 0.001600  | 0.000082  | 19.62   |
| 50,000      | 0.004846  | 0.000058  | 83.65   |
| 100,000     | 0.009743  | 0.000060  | 162.82  |
| 500,000     | 0.047446  | 0.000081  | 588.76  |
| 1,000,000   | 0.095926  | 0.000095  | 1005.86 |

**Среднее ускорение:** 310.17x  
**Максимальное ускорение:** 1005.86x  
**Минимальное ускорение:** 0.32x

## Анализ

- На малых размерах GPU медленнее из-за накладных расходов
- При увеличении размера данных GPU значительно быстрее
- Максимальное ускорение ≈ 60x

## Вывод

Параллельные вычисления эффективны при больших объемах данных
