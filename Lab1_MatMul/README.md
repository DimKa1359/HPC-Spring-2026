# Лабораторная работа 1: Перемножение матриц на CPU и GPU

## Описание
В лабораторной работе реализовано перемножение матриц с использованием CPU (NumPy) и GPU (CuPy). Проведен анализ производительности для матриц размером от 100×100 до 2000×2000.

## Реализация

### CPU
Используется `np.dot()` из библиотеки NumPy, которая вызывает оптимизированные BLAS-библиотеки (OpenBLAS/MKL), автоматически распараллеливающие вычисления на всех ядрах процессора.

### GPU
Используется библиотека CuPy:
- Данные приводятся к типу `float32` для оптимальной работы GPU
- `cp.asarray()` переносит данные в память GPU
- `cp.dot()` выполняет умножение с использованием cuBLAS — высокооптимизированной библиотеки линейной алгебры для GPU
- `cp.cuda.Stream.null.synchronize()` обеспечивает точность измерения времени

## Что распараллелено
- Само матричное умножение: каждый элемент результирующей матрицы вычисляется параллельно на тысячах ядер GPU
- Использование cuBLAS: библиотека реализует оптимальные алгоритмы для работы с памятью GPU и максимальной утилизацией вычислительных блоков

## Вывод

1. Для малых матриц (до 300×300) GPU неэффективен из-за накладных расходов на передачу данных между CPU и GPU.

2. Для средних матриц (400–1200) наблюдается нестабильное ускорение, связанное с особенностями загрузки памяти GPU.

3. Для больших матриц (от 1300×1300) GPU стабильно превосходит CPU с ускорением до 8.84×. Это объясняется массовым параллелизмом GPU, эффективной работой cuBLAS и оптимальным использованием памяти.

4. Вывод: применение GPU для перемножения матриц целесообразно при размерах матриц не менее 1300×1300, когда вычислительная нагрузка оправдывает накладные расходы на передачу данных.
