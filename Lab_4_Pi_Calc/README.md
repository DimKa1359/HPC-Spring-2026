# Лабораторная работа 4 — Вычисление числа π методом Монте-Карло на GPU (CUDA)

## Описание

Реализация метода Монте-Карло для вычисления числа π с использованием CUDA и библиотеки CURAND. Программа сравнивает время выполнения и точность результата на CPU и GPU.

Метод основан на отношении количества случайных точек, попавших в единичную окружность, к общему числу точек в квадрате [0,1]×[0,1]:

```
π ≈ 4 · (количество точек внутри окружности) / N
```

Точка (x, y) считается попавшей в окружность, если x² + y² < 1.

## Сборка

```bash
nvcc pi_calc.cu -o pi_calc -O2 -lcurand
```

## Запуск

```bash
./pi_calc <N>
```

**Параметры:**
| Параметр | Описание |
|----------|----------|
| `N` | Количество случайных точек |

**Пример:**

```bash
./pi_calc 100000000
```

## Реализация

### CPU-версия

Последовательная генерация N случайных точек с помощью `std::mt19937`. Для каждой точки проверяется условие попадания в единичную окружность. Время замеряется через `std::chrono::high_resolution_clock`.

### GPU-версия (CUDA)

- Случайные числа генерируются непосредственно на устройстве с помощью device API библиотеки CURAND (`curand_uniform`).
- Используются два CUDA-ядра: первое (`pi_kernel`) выполняет генерацию точек и частичную редукцию внутри каждого блока через разделяемую память; второе (`reduce_kernel`) суммирует результаты всех блоков в одно итоговое значение.
- Размер блока: 256 потоков, количество блоков: 256.
- Время выполнения GPU замеряется через `cudaEvent`.

## Пример вывода

```
CPU time: 2.847 s
Pi (CPU): 3.14158

GPU time: 0.031 s
Pi (GPU): 3.14161
```

## Анализ результатов

При малых значениях N (до ~100 000) GPU работает **медленнее** CPU — накладные расходы на инициализацию CUDA и CURAND перекрывают вычислительный выигрыш. Точка безубыточности находится в районе N = 1 000 000.

При дальнейшем росте N ускорение растёт почти линейно: при N = 100M достигает 530×, при N = 10B — 1718×. Это объясняется тем, что GPU время практически не увеличивается вплоть до N = 100M (~7 мс), после чего масштабируется плавно — вычисления не насыщают пропускную способность устройства на малых N.

Точность вычислений растёт с увеличением N для обоих методов. При N = 10 000 000 000 оба результата совпадают с эталонным значением π = 3.14159 до пяти значащих цифр.
